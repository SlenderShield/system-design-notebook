## 🧠 The Universal Design Checklist (Autopilot)

```
Requirements → Entities → Bottlenecks → Patterns → Tradeoffs → Evolution
```

| Step                    | LLD Focus                   | HLD Focus                   |
| ----------------------- | --------------------------- | --------------------------- |
| 1. Clarify Requirements | Use cases, actors           | Functional + Non-functional |
| 2. Define Core Objects  | Classes, relationships      | Services, data stores       |
| 3. Identify Bottlenecks | God objects, tight coupling | Read/write ratio, latency   |
| 4. Apply Patterns       | Strategy, Observer, Factory | Cache, Queue, Sharding      |
| 5. Defend Tradeoffs     | Why this pattern?           | Why this architecture?      |
